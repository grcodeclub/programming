Χειρισμός εξαιρέσεων

Κατά τη διάρκεια ανάπτυξης ενός προγράμματος, μπορεί να υπάρξουν περιπτώσεις, όπου δεν έχουμε τη βεβαιότητα ότι το συγκεκριμένο κομμάτι κώδικα θα δουλέψει σωστά, είτε επειδή μπορεί να προσπαθήσει να προσπελάσει πόρους που δεν υπάρχουν, είτε επειδή βγαίνει εκτός ορίων, ...

try { // κώδικας που θα δοκιμαστεί
	throw exception;}
catch (type  exception) {
	// κώδικας που θα εκτελεστεί σε περίπτωση εξαίρεσης
}
  1. Ο κώδικας στο μπλοκ try εκτελείται κανονικά
  2. Αν κάτι συμβεί ο κώδικας χρησιμοποιεί τη λέξη throw και μια παράμετρο για να προκαλέσει εξαίρεση
  3. Αν έχει συμβεί μια εξαίρεση, το μπλοκ catch εκτελείται λαμβάνοντας ως παράμετρο αυτή που περάστηκε από το throw

Η C++ διαθέτει προκαθορισμένες κλάσεις εξαιρέσεων στην std βιβλιοθήκη. Η std::exception είναι η βασική κλάση από την οποία μπορούν να κληρονομηθούν άλλες κλάσεις εξαιρέσεων.

std::runtime_errorΧρησιμοποιείται για να αναφέρει λάθη που συμβαίνουν κατά την εκτέλεση του προγράμματος και που δεν μπορούν να προβλεφθούν κατά τη διάρκεια του χρόνου μεταγλώττισης.Παραδειγμα
std::logic_erroρΧρησιμοποιείται για να αναφέρει λάθη λογικής που μπορεί να προβλεφθούν και να διορθωθούν κατά τον χρόνο ανάπτυξης του προγράμματος.Παραδειγμα
std::out_of_rangeΧρησιμοποιείται για να αναφέρει προσπάθειες πρόσβασης σε στοιχεία εκτός των ορίων μιας δομής δεδομένων (π.χ. πρόσβαση σε έναν δείκτη εκτός των ορίων ενός πίνακα).Παραδειγμα
std::invalid_argumentΧρησιμοποιείται για να αναφέρει μη έγκυρα επιχειρήματα που δίνονται σε μια συνάρτηση.Παραδειγμα
std::bad_allocΧρησιμοποιείται για να αναφέρει αποτυχία κατανομής μνήμης. Συνήθως πετιέται από τον τελεστή new όταν αποτυγχάνει να δεσμεύσει τη μνήμη που ζητήθηκε.Παραδειγμα
